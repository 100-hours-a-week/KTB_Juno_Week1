## 회고

### 신경써서 구현한 부분
- 2차 상속의 경우 처음에는 포지션을 기반으로 나누려고 했으나, 매니저의 경우 2차 상속을 받기가 모호하다고 생각하여 `Person` -> `KarasunoMember` -> `Player` / `Manager`로 나누게 되었다.
- grade의 경우에도 처음에는 `int`를 통해서 받는 걸로 처음에는 구현했었는데, 구현하다보니 println을 통해서 처음에 출력할 때 println("학년: " + grade + "학년"); 의 형태로 구현하게 되어서 `String`을 통해 받아서 "1학년", "2학년" .. 과 같이 받는 것이 더 효율적이라고 생각되어 `String` 타입으로 바꾸게 되었다.
- 매니저의 supportSkill을 경기 전력에 반영하려고 했으나, 단순히 karasunoPower에 그저 고정값을 더하는 방식이 될 것 같아 기능 확장보다는 스킬명 조회 기능으로 구현 범위를 줄였다.
### 구현 중 어려웠던 부분
- **클래스 다이어그램 작성 과정**에서의 어려움
    - 우선 상속의 화살표를 어디에서 어디로 그어야 하는지, 검정색 화살표로 긋는지, 아니면 빈 화살표로 긋는지 등도 헷갈렸고 상속이 아닌 관계는 어떤 선을 이용해서 어떻게 이어야 할 지가 어려웠다. 다행히 여러 자료를 찾아보면서 화살표를 긋기는 하였으나, 이 부분은 추후 더 스케일이 큰 프로젝트할 때는 대비해서 더 공부해야 할 부분이라고 느꼈다.
    - 현재 과제로 제출한 클래스 다이어그램도 조금 더 깔끔하고 가독성 좋게 그릴 수 있지 않을까라는 생각이 든다.
- 과제하기 전에 가장 우려했던 부분이 **자바 언어 사용(특히 문법)에 익숙하지 않음** 이었음.
    - 최대한 이해하고 간단하게 사용할 수 있는 println, scanner, 조건문을 위주로 최대한 아는 선에서 쉽게 구현하려고 노력하였다.
    - 상속, super()와 같은 부분은 클래스 다이어그램 작성 시에 개념적인 부분을 공부하여 머리로는 이해하였으나 막상 구현을 하려니 어떻게 시작해야 될 지 막막한 감이 있었다. 그런데 기존에 우선 되는 대로 짠 ~println이 난무하는..~ 코드에서 중복되거나 반복되는 코드들이 보였고, 이런 경우에 '상속을 쓸 수 있구나' 를 더 명확하게 느낄 수 있었다.
    - 처음에는 Player, Manager 클래스 모두에서 `this.school = "카라스노 고교"`와 같은 형식으로 매번 초기화했는데 구현을 하다보니 school의 경우 Player와 Manager의 공통 속성이라서 KarasunoMember 클래스에서 고정에서 넣는 형식으로 구현하였다.
